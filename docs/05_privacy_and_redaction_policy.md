# Privacy And Redaction Policy

## Purpose

This public documentation must communicate engineering and business value without exposing private operational data.

## Excluded From Public GitHub

Do not publish:

- Credentials
- API keys
- OAuth tokens
- App passwords
- Real email addresses
- Real email bodies
- Customer or supplier names
- Private inbox reports
- Trading logs
- Live config
- Runtime signal files
- Account data
- Broker data
- Token files
- Browser profiles
- Raw runtime memory files

## Allowed Content

Allowed materials include:

- Sanitized architecture descriptions
- Business workflow case studies
- Synthetic sample outputs
- Resume and interview snippets
- High-level diagrams
- Manually reviewed screenshots using synthetic data only

## Synthetic Data Rule

Any examples must be invented for demonstration. Use placeholder domains such as `example.test` and generic business scenarios such as supplier renewal, payment approval, meeting confirmation, and internal report updates.

## Review Checklist

Before publishing:

- Search for real email addresses
- Search for credentials and token-like strings
- Search for private names and organizations
- Search for account identifiers
- Confirm no logs or runtime output were copied
- Confirm no live configuration was copied
- Confirm AI-MT4 content is architecture-level only

## Principle

The documentation should show judgment, structure, and communication skills. It should not expose private data to prove technical ability.
